# Skill: Node-RED Patterns & Recipes

## What
This is the RECIPES skill — complete, ready-to-use flow patterns organized by use case.

## Why
- Most user requests map to well-known patterns (HTTP endpoint, MQTT subscriber, timer, routing, error handling)
- Without recipes, LLM reasons from first principles every time — slower, less reliable
- Consolidates pattern recipes + modularization patterns from flow-architecture

## Scope
- In scope: HTTP/REST patterns, MQTT/IoT patterns, data transformation, scheduling, message routing, error handling & resilience, rate limiting, join streams, subflow/modularization patterns
- Out of scope: Full node property listings (→ nodered-node-reference), build procedures (→ nodered-flow-builder)
