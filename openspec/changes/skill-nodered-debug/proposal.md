## Why

Testing and debugging Node-RED flows from the MCP requires a clear workflow: inject → observe debug output → identify problem → fix. The `inject-message` and `read-debug-messages` tools are powerful but require guidance on how to combine them effectively. This skill provides that workflow and explains how debug nodes should be configured for optimal observability.

## What Changes

- Add `nodered-debug` skill: a guide for the inject → debug → iterate workflow using MCP tools
