## Why

As flows grow, without deliberate organization they become unmaintainable: a single tab with hundreds of nodes, no labels, no documentation. The Node-RED "Developing Flows" guide provides best practices for structuring flows. An LLM-facing skill distills these practices into actionable guidance for when the LLM creates or refactors flows via MCP.

## What Changes

- Add `nodered-flow-architecture` skill: best practices for organizing Node-RED flows
